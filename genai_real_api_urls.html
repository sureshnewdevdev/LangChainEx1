Real Gen‑AI Chat API URLs – W3Schools Style Notes

In examples we often use a fake URL like https://api.example-ai.com/v1/chat. This page shows how to replace it with real URLs for OpenAI, Hugging Face, and a local (free) Ollama server.

1. Why there is no “free, no‑key” public URL

A URL like https://api.example-ai.com/v1/chat is only a placeholder. Real Gen‑AI providers require:

  • HTTPS endpoint (e.g. https://api.openai.com/...)
  • An API key in the request header
  • A model name (for example gpt-4.1-mini) in the JSON body
Important: There is no official, unlimited, “free & anonymous” Gen‑AI URL. For practice you can:
  • Use a provider’s free tier with an API key (OpenAI, Hugging Face, etc.)
  • Run a local model like Ollama (no key, runs only on your machine)

2. Real OpenAI Chat API URL

Official OpenAI chat completions endpoint (needs account + API key): 

POST https://api.openai.com/v1/chat/completions
Authorization: Bearer YOUR_OPENAI_API_KEY
Content-Type: application/json

2.1 Example Request Body

{
  "model": "gpt-4.1-mini",
  "messages": [
    { "role": "user", "content": "Say hello in one short sentence." }
  ],
  "max_tokens": 50,
  "temperature": 0.5
}

2.2 Python Example

import requests

API_URL = "https://api.openai.com/v1/chat/completions"
API_KEY = "YOUR_OPENAI_API_KEY"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

payload = {
    "model": "gpt-4.1-mini",
    "messages": [
        {"role": "user", "content": "Say hello from OpenAI in one sentence."}
    ]
}

response = requests.post(API_URL, headers=headers, json=payload)
print(response.json())

3. Hugging Face Inference API URL (free tier)

Hugging Face offers many open‑source models with a free usage tier. You still need a (free) account and a token, but calls go to a real URL: 

POST https://api-inference.huggingface.co/models/gpt2
Authorization: Bearer YOUR_HF_TOKEN
Content-Type: application/json

3.1 Simple Text Generation Example

import requests

API_URL = "https://api-inference.huggingface.co/models/gpt2"
HF_TOKEN = "YOUR_HF_TOKEN"

headers = {
    "Authorization": f"Bearer {HF_TOKEN}",
    "Content-Type": "application/json"
}

payload = { "inputs": "Explain vector database in one simple paragraph." }

response = requests.post(API_URL, headers=headers, json=payload)
print(response.json())
Note: Different models on Hugging Face may expect slightly different payload formats, but the URL pattern stays: https://api-inference.huggingface.co/models/<model-name>.

4. Local Free URL – Ollama

If you want a truly free per‑request URL for practice, you can run a model locally using Ollama. After installing Ollama and pulling a model (e.g. ollama pull llama3), you get a local HTTP API with no key required.

4.1 Ollama Chat Endpoint

POST http://localhost:11434/api/chat
Content-Type: application/json

4.2 Example Request Body

{
  "model": "llama3",
  "messages": [
    { "role": "user", "content": "Explain RAG in 3 bullet points." }
  ]
}

4.3 Python Example

import requests

API_URL = "http://localhost:11434/api/chat"

payload = {
    "model": "llama3",
    "messages": [
        {"role": "user", "content": "Explain RAG in 3 bullet points."}
    ]
}

response = requests.post(API_URL, json=payload)
print(response.json())
Good for lab: This is perfect for teaching in a classroom where internet is restricted. Every student can run a local model and hit http://localhost:11434/api/chat from Postman or Python.

5. How to replace https://api.example-ai.com/v1/chat in your code

In training material we used: 

API_URL = "https://api.example-ai.com/v1/chat"

5.1 Replace with OpenAI

# OLD (fake)
API_URL = "https://api.example-ai.com/v1/chat"

# NEW (real OpenAI)
API_URL = "https://api.openai.com/v1/chat/completions"

5.2 Replace with Local Ollama

# For completely free local testing (no key)
API_URL = "http://localhost:11434/api/chat"
Remember:
  • You must also set the correct headers (API key) and payload format.
  • The URL alone is not enough – check each provider’s docs for exact JSON fields.
