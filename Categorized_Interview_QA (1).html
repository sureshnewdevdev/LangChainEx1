1. Introduction & General

Q1. Give a brief self-introduction.
A. Start with your name, current role or education, overall experience, and strongest skills (Python, SQL, GenAI, Cloud). Then mention 1–2 key projects such as your Data Foundation project and GenAI E-Commerce chatbot, and close with your career goal (e.g., becoming a data/GenAI engineer).
Q2. What are the projects you have done?
A. Mention at least two: (1) a Data Foundation or Multi-Source Retail Data Integration project where you built ETL pipelines and a clean data layer; (2) a GenAI E-Commerce Chatbot project where you used LLMs, vector DB, and RAG for product Q&A. Briefly explain the goal, tech stack, and your role in each.
Q3. What do you understand by an algorithm?
A. An algorithm is a finite, step-by-step procedure or set of rules to solve a specific problem or perform a computation. It takes input, applies a sequence of deterministic or probabilistic steps, and produces output.
Q4. What do you understand by system architecture?
A. System architecture is the high-level design of a system that defines its components (UI, services, databases, APIs, queues, etc.), their responsibilities, and how they interact. It covers technology choices, data flow, integration points, and non-functional aspects like scalability and security.
Q5. Why should we hire you for this role?
A. Connect your skills (Python, SQL, GenAI, Cloud), your project experience (retail data foundation, GenAI chatbot), and your attitude (ownership, continuous learning, communication). Show that you can quickly start contributing to data/AI use cases relevant to their business.

2.2 SDLC & Scenario-Based Questions

Q9. Which SDLC model would you select to develop a project and why?
A. For data/GenAI projects, Agile or Scrum is usually best because requirements evolve as stakeholders see the data and model behaviour. Short sprints, frequent demos, and iterative improvements help refine metrics, prompts, and pipelines faster than a rigid Waterfall model.
Q10. After software deployment, if bugs are reported, how will you fix them?
A. Reproduce the bug, check logs and monitoring, identify root cause, create a small reproducible test case, fix the code or configuration, run unit/regression tests, deploy via proper CI/CD, and communicate clearly with stakeholders about impact and resolution.
Q11. How would you explain a complex problem to your teammate?
A. Break the problem into smaller pieces, use simple language and diagrams (data flow, sequence, architecture), relate to a familiar example, and confirm understanding by asking them to restate the solution or walk through a small example themselves.
Q12. How can you handle remote communication effectively?
A. Use clear written updates (email/Teams), maintain regular stand-ups, document decisions, over-communicate risks, share screen for live debugging, and use tools like Jira/Trello to track tasks so the team has a single source of truth.
Q13. Which tool would you use for communication with your teammates (Gmail, WhatsApp, Teams) and why?
A. For professional work, I prefer Teams/Slack or official email because they support channels, file sharing, and integrated task tracking. WhatsApp is informal and less controlled, so I use it only if the team or client explicitly prefers it.

2.3 Teamwork & Conflict Handling

Q14. How do you handle team conflicts?
A. I focus on facts and project goals rather than personal opinions, listen to both sides, clarify misunderstandings, and propose a data-driven or time-boxed experiment. If needed, I escalate to the lead with a solution-oriented mindset.
Q15. How do you handle written communication?
A. I keep emails and messages concise, structured with bullet points, mention clear action items and deadlines, and avoid ambiguous language. For important decisions, I always summarize in writing so everyone has the same understanding.

3. Python

3.1 Concepts & Advantages

Q16. Why choose Python over other languages?
A. Python has simple syntax, a huge ecosystem of libraries for data, AI and web (NumPy, Pandas, PyTorch, FastAPI, etc.), and strong community support. It lets you go from prototype to production quickly, which is ideal for data and AI projects.
Q17. What is special about Python compared to Java?
A. Python is dynamically typed and interpreted, which speeds up development and experimentation, especially in data science and AI. Java is statically typed and compiled, which can give performance and type safety advantages in large enterprise systems.

3.2 Core Syntax & Data Structures

Q18. What is the purpose of comments in Python?
A. Comments (using # or docstrings) explain what the code is doing, making it easier for others (and your future self) to read and maintain. They are ignored by the interpreter.
Q19. What is the self keyword in Python?
A. self is a reference to the current instance of a class. It is used inside methods to access instance variables and other methods. It is passed explicitly as the first parameter by convention.
Q20. What is a lambda function in Python?
A. A lambda is an anonymous, one-line function defined using the lambda keyword, typically used for short callback functions like sorted(data, key=lambda x: x[1]).
Q21. What does yield do in Python?
A. yield turns a function into a generator. Instead of returning all values at once, it yields one value at a time, allowing you to iterate lazily and handle large datasets efficiently.
Q22. How do you create a list of 10 numbers in Python?
A. For example: nums = list(range(1, 11)) creates a list from 1 to 10. Or nums = [i for i in range(10)] for 0 to 9.
Q23. Explain list vs tuple.
A. A list is mutable (you can add/remove/modify elements) and defined with []. A tuple is immutable (cannot be changed after creation) and defined with (). Tuples are often used for fixed collections of values.
Q24. What is the difference between list() and extend()?
A. list() is a constructor used to create a new list, usually from another iterable. extend() is a list method that appends all items from another iterable to an existing list in-place.
Q25. What is the difference between a list and a dictionary?
A. A list is an ordered collection indexed by integer positions. A dictionary is an unordered (insertion-ordered in modern Python) collection of key–value pairs, indexed by keys. Use lists for sequences and dictionaries for mapping relations.

3.3 Programs & Problem Solving

Q26. Write a Python program to check if a number is an Armstrong number.
A. An Armstrong number is equal to the sum of its own digits each raised to the power of the number of digits. Example logic: convert number to string, compute sum of digit**len(str), and compare with original number.
Q27. How would you check if a string is a palindrome in Python?
A. You can compare the string with its reverse: s == s[::-1]. If equal, it is a palindrome.
Q28. Swap two numbers without using a temp variable.
A. In Python you can use tuple unpacking: a, b = b, a. This swaps the values without an extra variable.

3.4 Testing & Sorting

Q29. What is unittest in Python?
A. unittest is Python’s built-in unit testing framework. It provides test case classes, assertions, setup/teardown methods, and test discovery to automate testing.
Q30. What are different types of testing you know?
A. Unit testing, integration testing, system testing, regression testing, acceptance testing, and performance testing. In Python projects, unit and integration tests are very common.
Q31. How do you perform sorting in Python?
A. You can use the built-in sorted(iterable) which returns a new sorted list or the list method list.sort() which sorts in-place. Both accept a key function and reverse flag.

4. SQL

4.1 Basics & Commands

Q32. What is SQL and why do we use it?
A. SQL (Structured Query Language) is used to store, manipulate, and retrieve data in relational databases. It provides a standard way to define tables (DDL) and query/update data (DML).
Q33. Why can’t we use any other language instead of SQL?
A. You can use ORM or APIs, but internally they still generate SQL for relational databases. SQL is optimized for set-based operations, joins, and aggregations, which are difficult to express efficiently with general-purpose languages alone.
Q34. Explain primary key and foreign key.
A. A primary key uniquely identifies each row in a table and cannot be NULL. A foreign key is a column that refers to the primary key of another table, enforcing a relationship between the two tables.
Q35. What is referential integrity?
A. Referential integrity ensures that a foreign key value always refers to an existing primary key in the parent table. It prevents orphaned records and keeps relationships consistent.

4.2 Data Types & DDL/DML

Q36. Explain char, varchar, and nvarchar.
A. char(n) stores fixed-length strings; it pads shorter values with spaces. varchar(n) stores variable-length strings up to n characters. nvarchar stores Unicode strings for multiple languages.
Q37. Explain DELETE, TRUNCATE, and DROP.
A. DELETE removes rows one by one and can use a WHERE clause; it can be rolled back. TRUNCATE quickly removes all rows from a table and usually cannot be rolled back in all DBs; structure remains. DROP removes the entire table (structure and data).

4.3 Clauses, Joins & Queries

Q38. What is the difference between WHERE and HAVING?
A. WHERE filters rows before grouping; it cannot use aggregate functions. HAVING filters groups after aggregation and can use aggregate functions like COUNT, SUM, etc.
Q39. Explain different types of joins.
A. INNER JOIN returns matching rows between tables, LEFT JOIN returns all rows from the left with matching right rows (or NULLs), RIGHT JOIN is the opposite, FULL JOIN returns rows that appear in either table, and SELF JOIN joins a table to itself.
Q40. What is a self join?
A. A self join is a join of a table with itself, typically using aliases, for hierarchical or comparative queries (e.g., managers and employees in the same table).
Q41. Explain the EXISTS operator and its alternative.
A. EXISTS returns TRUE if the subquery returns at least one row and is often used for semi-joins. An alternative is IN or using joins, although performance may differ depending on the database and query.
Q42. How do you find the 2nd highest salary in SQL?
A. One method: SELECT MAX(salary) FROM employees WHERE salary < (SELECT MAX(salary) FROM employees);. Another is using ROW_NUMBER or DENSE_RANK over ordered salaries.
Q43. How do you handle NULL values in SQL?
A. Use functions like COALESCE or ISNULL, and be careful with comparisons using IS NULL or IS NOT NULL. Aggregations usually ignore NULLs.
Q44. What is the difference between LIMIT and TOP?
A. LIMIT is used in MySQL/PostgreSQL at the end of the query to restrict the number of rows. TOP is used in SQL Server at the beginning of the SELECT, e.g., SELECT TOP 10 *.

5. GenAI, LLMs, Tokenization & RAG

5.1 Tokenization & Encodings

Q45. What is tokenization in NLP models?
A. Tokenization is the process of converting raw text into smaller units called tokens (words, subwords, or characters) that the model can process. Each token is then mapped to an ID and eventually to an embedding vector.
Q46. Will it work if you use the Mistral tokenizer with a LLaMA model?
A. Generally no. Each model is trained with its own tokenizer and vocabulary. Using a mismatched tokenizer will produce token IDs the model was never trained on, leading to poor or undefined behaviour.
Q47. What do you mean by encoding in models?
A. Encoding usually refers to converting text into numerical representations (token IDs or embeddings). The encoder part of a model transforms tokens into vectors that capture semantic and contextual information.

5.2 Embeddings & Model Training

Q48. What are embedding models and where do you use them?
A. Embedding models map text (or other data) into dense vectors in a semantic space. They are used for semantic search, recommendations, clustering, and as the basis for RAG workflows with vector databases.
Q49. What is LoRA (often written as Llora) in fine-tuning?
A. LoRA (Low-Rank Adaptation) is a parameter-efficient fine-tuning technique where small low-rank matrices are learned and added to the original weights instead of updating all model parameters, making fine-tuning cheaper and easier to store.
Q50. How do you evaluate your model performance in GenAI projects?
A. Use a mix of automatic metrics (BLEU/ROUGE for text, accuracy for classification), task-specific test sets, human evaluation for quality and helpfulness, and LLM-based evaluators to score correctness, faithfulness, and safety.

5.3 RAG & Vector Databases

Q51. What is RAG (Retrieval-Augmented Generation)?
A. RAG is a pattern where an LLM uses external knowledge retrieved from a data source (often a vector database) at query time. The retrieved context is added to the prompt so the model can give grounded, up-to-date answers.
Q52. What is a vector database and why is it used?
A. A vector database stores embedding vectors and supports similarity search (nearest neighbour queries). It is used in GenAI to find semantically similar documents or passages to a user query for RAG, recommendations, and clustering.
Q53. How are Pinecone and Chroma used in GenAI projects?
A. Pinecone is a managed vector database service; Chroma is an open-source local vector store. Both store embeddings with metadata and expose APIs to perform semantic search, typically used as the retrieval layer in RAG.
Q54. What is the difference between vector search and similarity search?
A. In practice, they often mean the same thing: searching by closeness in embedding space. Sometimes vector search refers to using dense vectors in general, while similarity search emphasises the similarity metric (cosine, dot product) used to rank results.

5.4 LangChain, Agents & LLM Families

Q55. What is LangChain?
A. LangChain is a framework that provides building blocks to connect LLMs with tools, data sources, and vector stores. It helps you build chains and agents for RAG, question answering, and workflow automation.
Q56. What are agents in LangChain?
A. Agents are components that decide which tool or action to take next based on the user input and intermediate results. They use an LLM as a reasoning engine to plan tool calls dynamically.
Q57. Claude vs GPT – what are some differences?
A. Both are powerful LLM families. GPT (OpenAI) and Claude (Anthropic) differ in training data, safety alignment, pricing, and context length. Claude is often praised for long-context reasoning and safety, while GPT is widely integrated and strong at coding and general tasks.
Q58. What are temperature, top-p, and top-k in LLMs?
A. Temperature controls randomness; higher values produce more diverse outputs. Top-k limits sampling to the top k most probable tokens. Top-p (nucleus sampling) chooses from the smallest set of tokens whose cumulative probability exceeds p.
Q59. What is a GAN?
A. A Generative Adversarial Network consists of a generator that creates fake data and a discriminator that tries to distinguish real from fake. Both are trained together, and over time the generator learns to produce realistic outputs.
Q60. What is Google Vertex AI?
A. Vertex AI is Google Cloud’s unified platform for building, training, and deploying ML and GenAI models. It provides managed notebooks, AutoML, model registry, pipelines, and integration with foundation models.

6. Cloud Computing & DevOps

Q61. What is cloud computing?
A. Cloud computing is the on-demand delivery of computing resources (servers, storage, databases, networking, AI services) over the internet with pay-as-you-go pricing, instead of owning physical hardware.
Q62. What are common cloud service models?
A. IaaS (infrastructure as a service), PaaS (platform as a service), and SaaS (software as a service). IaaS provides raw compute/storage, PaaS adds managed runtime environments, and SaaS provides ready-to-use applications.
Q63. What is a VPC (Virtual Private Cloud)?
A. A VPC is a logically isolated virtual network in the cloud where you can launch resources (VMs, databases) with your own IP range, subnets, route tables, and security groups.
Q64. What is a NAT Gateway?
A. A NAT Gateway allows instances in a private subnet to access the internet (for updates or APIs) while preventing the internet from initiating connections to those instances.
Q65. Explain elasticity vs scalability.
A. Elasticity is the ability to automatically scale resources up or down based on demand in near real time. Scalability is the capacity of a system to handle increased load by adding resources, either vertically or horizontally.
Q66. Object storage vs block storage – give examples.
A. Object storage (e.g., Amazon S3, Azure Blob) stores data as objects in buckets, ideal for unstructured data like logs and images. Block storage (e.g., EBS, Azure Disk) provides raw disk volumes attached to VMs, ideal for databases and file systems.
Q67. Azure vs AWS – how would you compare them?
A. Both offer similar core services (compute, storage, databases, AI). AWS is often seen as slightly ahead in breadth of services; Azure integrates deeply with Microsoft tools and enterprise environments. Choice depends on existing ecosystem and customer preferences.
Q68. What is cloud security in simple terms?
A. Cloud security covers identity and access management, network security, encryption, monitoring, backup, and compliance. It ensures only authorised users and services can access data and resources.

7. Combined Python + SQL Scenario

Q69. Write an SQL query to select a customer’s second purchase date.
A. You can use window functions, for example: use ROW_NUMBER() or DENSE_RANK() partitioned by customer and ordered by purchase date, then filter where rank = 2.
Q70. How would you implement the same logic using Python?
A. Load the data into Pandas, sort by customer and date, use groupby and nth(1) (0-based index) to get the second purchase row per customer, or use a custom loop if not using Pandas.
